# 改善タスク: browser-use E2E で見えたテストしやすさ改善

## 結論

今回の browser-use E2E では、打刻・従業員マイページ・管理画面の主要機能は一通り操作できました。
本番機能として直ちに修正が必要な不具合は見つかっていません。

一方で、E2E 自動化をより安定させるための改善候補があります。

```text
browser-use
  ↓ 操作
React 画面
  ↓ API
Hono サーバー
  ↓ 保存
SQLite テストDB
```

## 改善タスク一覧

- [x] シフト削除・シフト公開・シフト希望取り下げの確認を `window.confirm` からアプリ内モーダルへ置き換える
  - 理由: ブラウザ標準の確認ダイアログは browser-use / E2E で扱いが不安定になりやすい
  - 期待結果: 画面上の「確定」「取り消し」ボタンとして検証できる

- [x] 時刻入力欄に安定した検証用属性を追加する
  - 対象例: シフト編集の開始/終了時刻、店舗マスタの開店/閉店時刻
  - 理由: `type="time"` はブラウザや自動操作環境によって値変更の確認が難しい場合がある
  - 期待結果: E2E で「時刻を変更して保存された」ことを安定して確認できる

- [x] シフト時刻変更専用の Playwright E2E を追加する
  - 理由: browser-use では見え方の確認、Playwright では値保存の機械的確認、という役割分担ができる
  - 期待結果: シフト追加・時刻変更・保存後の再表示まで自動で検証できる

- [x] E2E 用の状態付き seed helper を用意する
  - 例: 未処理の修正申請が1件ある状態、下書きシフトが1件ある状態、提出済みシフト希望が1件ある状態
  - 理由: 手順の途中で状態が変わると、再テスト時にチェックリストと画面状態がずれやすい
  - 期待結果: 画面ごとのテストをいつでも同じ状態から始められる

## 優先度

1. `window.confirm` のアプリ内モーダル化
2. シフト時刻変更の Playwright E2E 追加
3. 時刻入力欄の検証用属性追加
4. E2E 用 seed helper 整備

## 実装済み

- シフト削除・週次公開・シフト希望取り下げは、ブラウザ標準確認ではなくアプリ内モーダルで確認できるようにしました。
- シフト編集と店舗マスタの時刻入力に、E2E で狙いやすい検証用属性を追加しました。
- `tests/e2e/fixtask.spec.ts` に、シフト時刻変更・削除・公開・希望取り下げの専用E2Eを追加しました。
- `tests/e2e/helpers.ts` に、管理者ログイン・従業員ログイン・対象画面を開く共通ヘルパーを追加しました。

## 今回作成したスキル

今回の確認手順は Codex スキル `browseruse-e2e-checklist` として作成しました。

できること:

- 画面数を理解する
- 全機能を画面ごとにリストアップする
- browser-use で一つずつ実操作する
- チェックが入らない項目を再調査して再実践する
- 最後に「不具合」と「テストしやすさ改善」を分けて報告する
