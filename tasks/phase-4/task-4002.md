# 認証 API (PIN/管理者)

## 背景

api-spec.md の Auth セクションを実装する。

## 追加するファイル

- `src/server/routes/auth.ts`
- `src/server/services/auth.ts` (PIN 検証、bcrypt、ロック処理、セッション発行)

## エンドポイント

- `GET  /api/auth/employees`: 退職者除外の従業員一覧 (id, name, kana, store_ids)
- `POST /api/auth/pin-login`: `{employee_id, pin}` 検証 → セッション発行 (5 分)
- `POST /api/auth/admin-login`: `{login_id, password}` 検証 → セッション発行 (2 時間)
- `POST /api/auth/logout`: セッション削除
- `GET  /api/auth/me`: 現在ユーザー

## 実装方針

- bcrypt.compare で PIN/PW 検証
- PIN 失敗時 `pin_fail_count++`、5 回到達で `lock_until = now + 5min` に設定し、`pin_fail_count` リセット
- ロック中ログインは 423 + `lock_until` 返却
- 成功時 `pin_fail_count = 0`、`sessions` 行作成、Cookie セット
- セッション ID は 32byte ランダム hex
- 管理者ログインは sliding (リクエスト毎に expires_at 延長) — middleware 側で実装
