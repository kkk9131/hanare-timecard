# シフト API

## 背景

シフト編成・公開・閲覧・希望提出を実装する。

## 追加するファイル

- `src/server/routes/shifts.ts`
- `src/server/services/shifts.ts`

## エンドポイント

- `GET    /api/shifts?store_id&from&to&status`
  - staff: status=published 強制
  - manager+: 全件
- `POST   /api/shifts` (manager+): 1 件追加 (status=draft)
- `PATCH  /api/shifts/:id` (manager+)
- `DELETE /api/shifts/:id` (manager+, draft のみ)
- `POST   /api/shifts/publish` (manager+): 範囲内 draft を published に
- `GET    /api/shifts/conflicts?store_id&from&to`: 同一従業員同日重複・人員不足検出
- `GET    /api/shift-requests` / `/api/shift-requests/me` / `POST` / `DELETE`

## 実装方針

- 同一従業員 × 同日 × 重複時間帯はアプリ層で検査 → 違反は 409
- 公開操作では `audit_logs` に before/after を記録
- 公開済みの編集も可能だが必ず audit に残す
- 人員不足判定: 「日付ごとに割当人数 < 1」の単純判定 (拡張は後フェーズ)
