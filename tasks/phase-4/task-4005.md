# 修正申請 API と監査ログ

## 背景

修正申請の作成・承認・却下、および audit_logs の閲覧 API を実装する。

## 追加するファイル

- `src/server/routes/corrections.ts`
- `src/server/routes/audit.ts`
- `src/server/services/corrections.ts`
- `src/server/middleware/audit.ts` (汎用 audit ヘルパ)

## エンドポイント

- `GET  /api/corrections?status&store_id` (manager+)
- `GET  /api/corrections/me` (staff)
- `POST /api/corrections` (staff): 申請作成
- `POST /api/corrections/:id/approve` (manager+): 承認 → 打刻反映 + audit
- `POST /api/corrections/:id/reject` (manager+): `{review_comment}` 必須
- `GET  /api/audit?from&to&actor_id&action&page&limit` (admin)

## 承認時の挙動

- 対象 punch がある場合: 既存行を更新 (`punched_at`, `punch_type` を書き換え) し、`source='correction'`、note に申請 ID を残す
- 対象 punch が null (打刻漏れ) の場合: 新規挿入
- before/after を JSON 文字列化して `audit_logs` に挿入
- ステータスを `approved` に、`reviewer_id`, `reviewed_at` をセット

## 制約

- audit_logs の DELETE エンドポイントは実装しない
- 申請者本人と異なる打刻への申請は 403
