# Hono サーバ起動とセッション基盤

## 背景

Hono アプリの起動骨格、Cookie セッションミドルウェア、`/api/system/health` を実装する。

## 追加/変更するファイル

- `src/server/index.ts`: `serve({ fetch: app.fetch, hostname: '0.0.0.0', port })` で起動。LAN 向け IP を起動時にコンソール表示
- `src/server/app.ts`: Hono アプリ本体
- `src/server/middleware/session.ts`: Cookie `hanare_sid` を読み、`sessions` テーブルから検証して `c.set('user', ...)`
- `src/server/lib/crypto.ts`: ランダム ID 生成、HMAC 署名
- `src/server/routes/system.ts`: `/api/system/health`
- `src/server/middleware/error.ts`: 統一エラーレスポンス

## 実装方針

- セッション TTL: PIN ログイン 5 分、admin ログイン 2 時間 (sliding 更新)
- Cookie 属性: HttpOnly, SameSite=Lax, Path=/, Secure は環境変数 `HANARE_TLS=1` の時のみ
- 静的配信: 本番は `dist/client` を `serveStatic` で
- ポート: env `PORT` (default 3000)
- 起動時に `migrate.ts` を内部で呼ぶ (DB 初期化)
