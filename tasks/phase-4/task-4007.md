# 従業員と店舗マスタ API

## 背景

管理者が従業員・店舗を CRUD するための API を実装する。

## 追加するファイル

- `src/server/routes/employees.ts`
- `src/server/routes/stores.ts`
- `src/server/services/employees.ts`

## エンドポイント

- `GET  /api/stores` (認証)
- `POST /api/stores` (admin)
- `PATCH /api/stores/:id` (admin)
- `GET  /api/employees?store_id&include_retired` (manager+)
- `POST /api/employees` (admin): name, kana, role, login_id?, password?, pin, hourly_wage, hire_date, store_ids[]
- `PATCH /api/employees/:id` (admin)
- `POST /api/employees/:id/pin` (admin): PIN リセット
- `POST /api/employees/:id/retire` (admin): retire_date 設定

## 実装方針

- POST/PATCH は audit_logs に記録
- bcrypt ハッシュ化
- employee_stores の差分更新
- 退職処理は論理削除のみ (DELETE エンドポイントなし)
