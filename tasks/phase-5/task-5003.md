# 打刻フロー画面 K01-K04

## 背景

共用端末向けの打刻フロー (名前選択 → PIN → 打刻ボード → 完了) を実装する。最重要 UX。

## 追加するファイル

- `src/client/routes/PunchTop.tsx` (K01: 従業員選択タイル一覧)
- `src/client/routes/PunchPin.tsx` (K02: PIN 入力)
- `src/client/routes/PunchBoard.tsx` (K03: 出勤/退勤/休憩開始/休憩終了)
- `src/client/routes/PunchDone.tsx` (K04: 完了表示 + 5 秒で自動的に K01)
- `src/client/api/punches.ts`
- `src/client/api/auth.ts`

## 実装方針

- K01: 上部に縦組ロゴ「雀庵 / はなれ」、中央に BigClock、下部に EmployeeTile グリッド (5 列、kana 順)。店舗フィルタタブ
- K02: 選択した名前を表示 + PinPad。誤 PIN 時「申し訳ございません〜」。ロック時は丁寧文 + lock_until までのカウントダウン
- K03: 現在状態に応じてボタンを disable (working なら「出勤」disabled)。退勤時は確認ダイアログ「本当に退勤しますか？(HH:MM)」
- K04: 「{name}さん、お疲れさまです。{action}を記録しました（HH:MM）」を大きく表示、5 秒後 logout + K01 へ
- 全画面 ShojiTransition で切替
- 打刻状態は `/api/punches/me/state` で取得

## 制約

- 打刻ボタンは 1 画面 1〜2 個に絞る (UX 方針)
- 文字は大きく (見出し 28-32px、ボタン文字 20px+)
- エラー表示は赤地ではなく朱の差し色 + アイコン
