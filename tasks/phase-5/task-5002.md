# 和モダンスタイルと共通コンポーネント

## 背景

`agent-docs/ui-design.md` の和モダンスタイルガイドを実装する。CSS variables を `tokens.css` に定義し、共通コンポーネントを実装する。

## 追加するファイル

- `src/client/styles/tokens.css` (ui-design.md のカラー・タイポ・寸法)
- `src/client/styles/global.css` (リセット + body 背景 sumi-900 + フォント設定 + 背景の微細木目 SVG)
- `src/client/styles/wood-grain.svg` (もしくは CSS で生成した noise pattern)
- `src/client/components/WashiCard.tsx`
- `src/client/components/SumiButton.tsx` (variant: primary/ghost/danger)
- `src/client/components/BigClock.tsx` (1 秒更新, tabular-nums, 明朝)
- `src/client/components/EmployeeTile.tsx`
- `src/client/components/PinPad.tsx` (10 キー + クリア + 決定, 各キー 56×56 以上)
- `src/client/components/ShojiTransition.tsx` (横スライド, prefers-reduced-motion でフェード)
- `src/client/components/AndonHover.tsx` (ホバーで微発光)
- `src/client/components/StatePill.tsx`
- `src/client/components/AppShell.tsx` (左サイドバー + メイン, 縦組ロゴ)

## 実装方針

- 全部 CSS Modules or プレーン CSS (Tailwind は導入しない、和モダンの細やかさのため)
- アクセシビリティ: aria-label 必須、フォーカスリング kincha-400
- フォントは Google Fonts の Noto Serif JP / Noto Sans JP を CSS で読み込み (オフライン優先のため必要なら同梱)
- iPad 縦 (768×1024) で破綻しないこと
- ボタンの最小タップサイズ 56×56
