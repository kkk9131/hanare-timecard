# DB スキーマとマイグレーション

## 背景

`agent-docs/data-model.md` の DDL を Drizzle ORM スキーマで定義し、初回マイグレーションを生成する。

## 追加するファイル

- `src/server/db/schema.ts`: stores, employees, employee_stores, time_punches, shifts, shift_requests, correction_requests, audit_logs, sessions, work_days
- `src/server/db/client.ts`: better-sqlite3 + drizzle インスタンス。`HANARE_DB_PATH` 環境変数 (デフォルト `data/hanare.db`)
- `drizzle/0000_init.sql`: drizzle-kit generate の出力
- `scripts/migrate.ts`: `drizzle/` 配下の SQL を順次適用 (`__drizzle_migrations` テーブル管理)
- `drizzle.config.ts` の正しい設定

## 実装方針

1. data-model.md の各テーブルを `sqliteTable` で定義
2. CHECK 制約・インデックスは `sql\`...\`` で記述するか drizzle のヘルパで
3. `npx drizzle-kit generate` でマイグレーション SQL 生成
4. `scripts/migrate.ts` 実装 (better-sqlite3 で SQL 実行 + applied 記録)
5. `npm run migrate` で `data/hanare.db` が作成されること
