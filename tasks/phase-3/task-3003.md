# 共有型と zod スキーマ

## 背景

サーバ・クライアントで共有する型と API リクエスト/レスポンスの zod スキーマを定義する。`agent-docs/api-spec.md` のすべてのエンドポイントを網羅する。

## 追加するファイル

- `src/shared/types.ts`: `Role`, `PunchType`, `ShiftStatus`, `Store`, `Employee`, `TimePunch`, `Shift`, `CorrectionRequest`, `AuditLog`, `WorkDayRow` 等
- `src/shared/schemas.ts`: zod スキーマ
  - `pinLoginSchema`, `adminLoginSchema`
  - `createPunchSchema`
  - `createShiftSchema`, `updateShiftSchema`, `publishShiftSchema`
  - `createCorrectionSchema`, `reviewCorrectionSchema`
  - `createEmployeeSchema`, `updateEmployeeSchema`
  - `createStoreSchema`, `updateStoreSchema`
  - `exportQuerySchema`

## 実装方針

- zod スキーマから型を `z.infer` で導出し、極力重複定義を避ける
- 日付は 'YYYY-MM-DD'、時刻は 'HH:MM'、unix ms は number で扱う
- PIN は `/^[0-9]{4,6}$/`
- ロールは zod enum
