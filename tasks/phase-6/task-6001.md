# バックアップと起動スクリプト

## 追加するファイル

- `scripts/backup.ts`: SQLite `VACUUM INTO 'data/backups/hanare-YYYYMMDD-HHmm.db'`、30 世代超を削除
- `start.sh`: `#!/bin/bash`, `npm install` (idempotent) → `npm run build` → `node dist/server/index.js`
- `start.sh` 内で起動時に `node scripts/backup.ts` を呼ぶオプション (env `HANARE_BACKUP_ON_START=1`)
- `scripts/backup.cron.example` (launchd plist or crontab エントリの例)
- 管理画面からの即時バックアップ用に `POST /api/system/backup` を実装 (admin)

## 実装方針

- backup.ts:
  1. `data/backups/` を作成
  2. better-sqlite3 で `db.exec("VACUUM INTO '...'")` (絶対パス)
  3. `data/backups/hanare-*.db` の glob を mtime 降順でソートし 31 番目以降を fs.unlink
  4. 結果をログ + 戻り値で返す
- `start.sh` は実行権限 755
- README にバックアップの自動化手順 (launchd/cron) を記載するのは task-6002 で
