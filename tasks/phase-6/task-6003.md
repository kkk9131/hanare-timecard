# E2E スモークテスト

## 背景

主要フローを 1 本の Playwright スクリプトで通すスモークを用意する。

## 追加するファイル

- `playwright.config.ts`
- `tests/e2e/smoke.spec.ts`
- `package.json` に `test:e2e` スクリプト追加

## テストシナリオ

1. アプリ起動 (`webServer` 設定で `npm run start` を起動)
2. `/` を開く → 従業員タイル一覧が見える
3. シードの staff を選択 → PIN 入力 → 打刻ボード
4. 「出勤する」を押す → K04 で「出勤を記録しました」表示
5. 自動 logout 後 K01 に戻る
6. 同 staff で再ログイン → 「退勤する」を押す → 確認 → 完了
7. ブラウザのコンテキストを切り替え `/admin/login` で `oyakata` ログイン
8. ダッシュボードで「現在勤務中の従業員 0 名」または該当者表示を確認
9. `/admin/exports` を開き「今月をエクスポート」ボタンを押下 → ダウンロードイベントを capture
10. ダウンロードファイルの拡張子が `.xlsx` でサイズ > 0

## 実装方針

- Playwright `download` イベントでファイル取得
- 各ステップにアサーションを 1 つ以上
- 失敗時 trace を保存
