## completed

- `src/server/services/shifts.ts` 新規
  - `findConflicts(emp, date, start, end, excludeId?)`: 同一従業員・同日の時間帯重複を判定（境界接触は非重複）
  - `createShift`: 重複時 `kind:"conflict"`、初期 status=draft、audit_logs に `shift.create` を記録
  - `updateShift`: 自分自身は除外して再衝突検査、変更前後を audit に保存（公開済みも変更可、必ず履歴に残る）
  - `deleteShift`: draft 限定。published には `kind:"invalid"` を返す
  - `publishShift`: 単一シフトを draft→published、`shift.publish` audit
  - `publishShifts(storeId, from, to, actor)`: 範囲一括公開
  - `listShifts(query)`: store/employee/from/to/status フィルタ、date+start_time 昇順
  - `detectConflicts(storeId, from, to)`: 重複ペアと「割当 0 名の日 = 人員不足」を返す
  - `createShiftRequest` / `listShiftRequests` / `getShiftRequest` / `deleteShiftRequest`
- `src/server/routes/shifts.ts` 新規
  - GET /api/shifts: スタッフは `status=published` かつ `employee_id=自分` を強制
  - GET /api/shifts/conflicts (manager+)
  - POST /api/shifts (manager+): conflict→409, invalid→400
  - PATCH /api/shifts/:id (manager+): conflict→409
  - DELETE /api/shifts/:id (manager+): published 削除は 409
  - POST /api/shifts/publish (manager+)
  - GET /api/shift-requests (manager+) / GET /me (auth) / POST (auth) / DELETE :id (本人 or manager+)
- `src/server/app.ts`: shiftsRoutes / shiftRequestsRoutes をマウント
- vitest 13 ケース (`tests/unit/shifts.service.test.ts`)
