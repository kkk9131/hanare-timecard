## 未完了 / 既知ギャップ (known_gaps)

- 退職処理は「今日」を retire_date に固定。日付指定 UI は未実装 (チケット記述では確認ダイアログのみ言及)。
- フォームバリデーションはサーバに委ね、エラー時は汎用 Toast 表示のみ。フィールド単位のエラー強調は未実装。
- フォーカストラップは「初期フォーカスのみ」で、Tab で背景に抜ける可能性あり。簡素実装で運用。
- ui-design.md の `<EmployeeFormModal>` `<StoreFormModal>` 別ファイル切り出しはせず、route 内に inline で記述。後に再利用が必要なら抽出可。
- 監査ログ確認・権限境界 (manager は employees/stores にアクセスできない) は AuthGuard 側で担保されている前提。

## 重要判断

- Modal は別途 Floating UI / Headless UI 等を導入せず、最小 100 行の自前実装。和モダンの章番号 eyebrow + 「閉」漢字クローズボタンを表現するため、汎用ライブラリより自前を選択。
- biome の `useKeyWithClickEvents` 警告対策で backdrop クリック検知を `<button class="wa-modal__scrim">` に変更 (tabIndex=-1, aria-label="閉じる")。クリック領域は absolute 全面。
- correctionSchema を後方互換のため optional 拡張 (既存 dashboard が `id/employee_id/status` のみ参照)。新フィールド追加は破壊的変更なし。
- updateEmployee の login_id は `string | null` を許容 (サーバ schema 準拠)。
