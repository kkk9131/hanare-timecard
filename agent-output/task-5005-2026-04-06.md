## 実装サマリ

### 追加ファイル

**API ラッパ**

- `src/client/api/admin.ts` — 管理画面用エンドポイントのまとめ。
  - `adminLogin({login_id, password})` → POST /api/auth/admin-login (zod パース)
  - `listStores()` → GET /api/stores
  - `listPunches({store_id, employee_id, from, to})` → GET /api/punches
  - `listShifts({store_id, from, to, status})` → GET /api/shifts
  - `listCorrections({status, store_id})` → GET /api/corrections (404/501 は空配列フォールバック。phase 6 で実装予定)

**コンポーネント**

- `src/client/components/StoreSwitcher.tsx` / `.css` — 店舗切替タブ。fieldset + legend (sr-only) で a11y、aria-pressed で選択状態を色だけに頼らず表示。雀庵 / 雀庵はなれ / (任意) 全店舗。
- `src/client/components/AdminLayout.tsx` / `.css` — 管理画面共通レイアウト。AppShell sidebar スロットに 7 項目ナビ (ダッシュボード/シフト/修正申請/従業員/店舗/エクスポート/監査ログ) を配置。NavLink で active 状態を kogecha-700 + 金茶下辺で示す。明朝書体。

**ルート**

- `src/client/routes/AdminLogin.tsx` / `.css` — A00。
  - 中央配置の WashiCard に明朝見出し「雀庵 管理画面」+ ADMIN LOGIN eyebrow。
  - login_id / password input (`required`, autoComplete=username/current-password)。
  - 401 → 「ログイン ID またはパスワードが違います。今一度ご確認ください。」 / 423 → 「5 回連続で誤入力されたため、しばらくの間ロックされています…」 / その他 → 通信失敗の丁寧語メッセージ。
  - useMutation 成功時に react-query キャッシュを invalidate して `/admin` へ replace navigate。
  - 背景に kincha + shu のラジアルグラデーションで行灯の気配。
- `src/client/routes/AdminDashboard.tsx` / `.css` — A01。
  - StoreSwitcher で店舗フィルタ ("all" or store id)。
  - 3 枚のメトリック WashiCard:
    1. 「現在勤務中」 — 当日 GET /api/punches を 30 秒ポーリング、employee 単位で最終打刻種別から working / on_break を算出。72px 明朝大数字。
    2. 「未処理の修正申請」 — listCorrections({status:"pending"})。phase 6 まで 0 件表示でも壊れない。
    3. 「シフト充足率」 — 当週月〜日の published shift を取得し、日数カバレッジを % で表示 (公開枠数 / 対象日数併記)。
  - 「ショートカット」 WashiCard で エクスポート / 従業員 / 監査ログ への直行リンク。
  - 各カードに StatePill (success/warning/neutral) を併記し色だけに頼らない状態表示。

### 設計ハイライト

- **A00 と A01 の対比**: A00 はカードを画面中央に置く一点透視、A01 はサイドバー + 3 列メトリックの安定構成。両者で `WashiCard` の eyebrow + title API を使い回し統一感を保つ。
- **明朝の階層**: ロゴ (Logo) → 見出し (Heading) → メトリック数値 (72px font-mincho, tabular-nums) と 3 段階で品の連続性を作る。
- **店舗切替の a11y**: fieldset/legend + aria-pressed の二重表現。biome useSemanticElements / useAriaPropsSupportedByRole に準拠。
- **未実装エンドポイントへの保険**: `listCorrections` は 404/501 を空配列に丸めるので、修正申請 API が phase 6 で実装される前でもダッシュボードを壊さない。
- **iPad 縦/横対応**: dashboard grid は 1024px 以下で 2 列、640px 以下で 1 列。サイドバーは 900px 以下で上部バーへ折返し (AppShell が責任)。
