## known_gaps

- 5 回連続失敗 → 423 lock の end-to-end 検証は curl では未実施 (条件分岐単体は同じコードパス、unit test を後続フェーズで足すと安心)
- pin-login の `not_found` (退職者) ケースは 401 invalid_pin として返している (api-spec のサンプルが invalid_pin/locked のみ言及しているため整合)
- `verifyPin` は bcrypt 同期 API (`compareSync`) を使用。Node の event loop 観点では好ましくないが、ローカル kiosk 用途なので許容範囲。非同期化したい場合は後続フェーズで対応
- pin-login 試行時の 5xx 等のエラーハンドリングは Hono の global errorHandler 任せ
