## 設計上の判断

- backup ロジックを `src/server/lib/backup.ts` に置いた理由: `tsconfig.server.json` の rootDir が `src` で、
  サーバから `scripts/` を import すると rootDir 違反になるため。`scripts/backup.ts` は `src/server/lib/backup.ts` を
  re-export するだけの薄いラッパとした
- VACUUM INTO の絶対パスは SQL リテラルとしてシングルクォートをエスケープして埋め込み (better-sqlite3 はパラメータバインドを VACUUM 系で受け付けないため)
- 起動時 backup は失敗してもサーバ起動を止めないようガード
