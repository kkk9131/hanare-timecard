## 重要判断 (decisions)

- drizzle 0.45 系の `sqliteTable` 第 3 引数は「コールバック → 配列」形式 (`(table) => [...]`) を採用 (新しい API)
- CHECK 制約は `check(name, sql\`...\`)` で記述 (drizzle 0.31+)
- `client.ts` で module load 時に `createSqlite()` を呼び DB ディレクトリを自動作成。これにより `import { db }` 側で事前準備不要
- マイグレーション適用は自前実装 (drizzle-orm/better-sqlite3/migrator を使わず) で `drizzle/*.sql` を直接実行。理由: 最小依存・スクリプト側で動作が見える・`__drizzle_migrations` テーブル名は drizzle-kit 側と一致
- statement 区切りは drizzle-kit の `--> statement-breakpoint` マーカーで split (標準フォーマット)
