## A06 エクスポート

- ヒーロー: 「今月をエクスポート」大型 SumiButton (xlsx ワンクリック)
- 期間: クイック (今月 / 先月 / 今四半期) + カスタム date input (from/to)
- 店舗: 全店舗 / 雀庵 / 雀庵はなれ の chip 群 (`<fieldset>` + aria-pressed button)
- 形式: xlsx / csv の 2 大型カード
- サマリ + ファイル名プレビュー (`hanare-{store|全店舗}-{period}.{ext}`)
- ダウンロードは `<a href download>` を生成して click → ブラウザ標準の保存ダイアログ
- 完了後 `Toast` で「{filename} を保存しました」
- URL は `/api/exports/period.xlsx?from=&to=&store_id=` (server 側で実装済み, api-spec エイリアス `/api/exports/xlsx` も存在するが period.\* の方を採用)

## A07 監査ログ

- フィルタ: 期間 (from/to) / 行為者 (employees から select) / アクション (固定 13 種)
- リスト: 縦 timeline (時刻 → ドット → 本文) で `before_json` / `after_json` を JSON.parse して
  変更フィールド diff を 6 件まで表示。`from → to` を朱色取り消し線 + 通常色で表現。
- 作成 / 削除のみ (片側 null) は単行プレーン表示
- ページング: limit 50 固定、`offset` を `次の巻 / 前の巻` で前後移動。
  「巻 N」を eyebrow 表示 / `{offset+1}–{offset+len}` を action slot に表示
- フィルタ変更で offset を 0 にリセット
- アクション種別ごとに dot/text の色を 4 段階 (neutral / create / warn / delete) で出し分け

## API クライアント追加

```ts
buildExportUrl({ format: "xlsx", from, to, store_id });
// → /api/exports/period.xlsx?from=...&to=...&store_id=...

listAudit({ from, to, actor_id, action, limit, offset });
// → { logs: AuditLog[], limit, offset }
```

`AuditLog` schema は server 実装 (src/server/routes/audit.ts) に合わせて
`actor_id`, `entity_type`, `entity_id`, `before_json`, `after_json`, `occurred_at` を nullable で受ける。

## アクセシビリティ配慮

- 全 chip / format / quick ボタンは `<button type="button" aria-pressed>` ベース
- グルーピングは `<fieldset>` + `<legend className="sr-only">` で意味付け
- date input は `color-scheme: dark` でダークモードでも視認性確保
- 期間 invalid 時はダウンロード disabled + warn メッセージ表示
