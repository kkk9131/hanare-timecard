## known_gaps

- audit_logs の actor_id にシステムユーザー (1) を渡しているが、店主アカウントが employees の id=1 と一致しない場合はマイグレーション側の前提に依存する。今回は seed 実装上 admin=id1 でテスト pass
- POST /api/stores の UNIQUE 違反検知は better-sqlite3 のエラー文字列マッチに依存している。Drizzle/sqlite のメッセージが将来変わる可能性は残る
- updateEmployee の role 変更時に login_id/password の整合性チェック（staff→manager 昇格時に credential 不在ケース）は実装していない。チケット scope 外と判断
- listEmployees の store_ids フィルタはアプリ層で行っており、件数が大きい場合は join に最適化する余地あり
