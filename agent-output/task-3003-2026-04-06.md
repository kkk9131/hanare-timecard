## known_gaps

- `User` 型は data-model.md 通り Employee.role に統合する設計のため別型は作っていない。
  今後 `login_id + password` 経路で増えるフィールドがあれば `Employee` を拡張する
- `exportQuerySchema` の `format` (xlsx/csv) はルート URL (`/api/exports/xlsx`) で分岐する
  api-spec 設計のため query には含めていない
- `pinSchema` は `/^[0-9]{4,6}$/` と仕様通り。max/min だけでなく regex で縛っている
- `note` 等の任意文字列の max は実装上妥当な値 (256〜1024) を決め打ちした
  (spec には明記がないため、DB 制約追加時に調整要)
- `listEmployeesQuerySchema.include_retired` は string/boolean どちらでも受ける
  (クエリは string 前提だが client 直呼び出しにも対応)
