## 重要判断

- **vite proxy bypass の追加**: 既存設定のままだと `import { ... } from "../api/punches"` を Vite が `/api/punches.ts` URL として fetch し、proxy に飲まれてバックエンド 404 になるバグがあった。bypass で `/api/*.tsx?|css|map|js` をローカルに通す形に変更。既存の `/api/auth/employees` 等の本物の API リクエストは引き続き proxy 経由で動作することを confirm 済み。
- **kiosk.tsx を 4 ファイルへの再エクスポートにした**: チケットは `PunchTop.tsx` 等 4 ファイルを要求しているが、既存 `App.tsx` は `routes/kiosk.tsx` から 4 ページを import している。App.tsx を書き換えるより re-export の方が変更面積が小さい。
- **`auth.ts` のスキーマを書き換えた**: AuthGuard が依存している `fetchMe` の戻り値型 (`{id,name,role}`) はサーバの実装 (`{employee, session_expires_at}`) と一致しておらず、PIN ログイン後に `/punch/board` に進めない (毎回 `/` にリダイレクトされる) 致命的バグがあった。task-5003 の前提条件 (K03 が表示できること) のため修正必須と判断。
